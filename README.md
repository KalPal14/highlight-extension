# Введение

Этот проект — монорепозиторий с несколькими сервисами и клиентским приложением, созданный для демонстрации моих навыков в бэкенд и фронтенд разработке. Он включает в себя два Express сервиса, NestJs сервис, браузерное расширение для chrome и firefox на React. Функционал не зависящий от бизнес логики конкретного преложения, который используется или может быть использован в нескольких приложениях вынесен библиотеки.

#### Ключевые технологии:

- Backend: NestJS, Express.js, Prisma, TypeOrm
- Frontend: React
- Управление проектом: Nx, Docker
- Тестирование: Jest, Supertest

# Локальное разворачиванию и запуск приложений

### 1. Клонирование приложения с git hub

```
git clone https://github.com/KalPal14/focus-learn-flow.git

cd focus-learn-flow/
```

### 2. Установка npm версии 10.8.1 и node версии 20.16.0

Рекомендую использовать nvm

Для установки nvm воспользуйтесь командой

```
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.2/install.sh | bash
```

или

```
wget -qO- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.2/install.sh | bash
```

затем

```
export NVM_DIR="$([ -z "${XDG_CONFIG_HOME-}" ] && printf %s "${HOME}/.nvm" || printf %s "${XDG_CONFIG_HOME}/nvm")"
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh" # This loads nvm
```

Если возникли проблемы, посетите раздел [Installing and Updating](https://github.com/nvm-sh/nvm?tab=readme-ov-file#installing-and-updating), там вы сможете найти подробную инструкцию

Далее установите нужную версию npm и node командой

```
nvm install 20.16.0
```

Проверте версию node, в консле вы должны увидеть v20.16.0

```
node -v
```

Если это не так, запустите команду

```
nvm use 20.16.0
```

nvm установит npm нужной версии автоматически, вы можете проверить это, запустив команду. В консоле вы должны увидеть 10.8.1

```

```

# Архитектура и технические решения для монорепозитория

Проект представляет собой монорепозиторий слабо связанных между собой приложений, со стандартной архитектурой в которой все приложения находятся в папке `/apps`, а общий код не завязанный на бизнес логике одного конкретного приложения вынесен в папку `/libs`

### Обоснование связки npm workspaces + nx

Есть несколько путей построения монорепозитория. Их можно разделить на три типа.

- "Аутсорсный". Это когда почти всю конфигурацию монорепозитория, приложений и библиотек берёт на себя выбранный разработчиком сторонний фреймвёрк или библиотека для постороения монорепозиториев. Самые популярные это Nx и Lerna. Такой подход существенно упрощает и ускоряет процесс настройки монорепозитория, минимизирует требования к знаниям и умениям разработчика настраивать проекты, но лишает гибкости.
- Полностью кастомизированый. Когда разработчик сам настраивает каждое отдельное приложение, библиотеку и связь между ними с помошью npm, yarn vite workspaces. Это даёт максимальную гибкость, но требует от разработчиков больше знаний и времени на конфигурацию пмонорепозитория. Лишает кучи встроенных в Nx и Lerna фич: удобное управление одним или несколькими приложениями одновременно, встроенное кэширование (что заначительно ускоряет сборку проекта), графы зависимовстей между приложениями, инструменты для обнаружения циклических зависимостей и много что ещё.
- Гибридный. Когда основная конфигурация приложжений и библиотек происходит с помошью nx, yarn или vite workspaces, а Nx или Lerna присутствуют только на самом верхнем уровне. Такой подход всё ещё требует от разработчика времени и навыков. Но не лишает гибкости. Добавляет основные преймущества использования Nx или Lernа которых лишены разработчики выбравшие полностью кастомизированный вариант (я их перечислял в предыдущем пункте).

Учитывая все плюсы и минусы трёх описанных мною подходов, я решил остановится на третьем. На то есть несколько причин.

- Проект изначально не задумывался как монорепозиторий. Он плавно вырос из простого express приложения `highlight-extension` ([коммит](https://github.com/KalPal14/focus-learn-flow/commit/c2c426b7e8c17e5fe895addc99a99787f97ffa5a)) в котором уже пресутствывала базовая настройка отличная от того, как nx или lerna настраивает express приложения. По этому более органичное расширение могли предоставить только гибридный или полностью кастомный подходы.
- Идея монорепозитория которая у меня родилась в прецессе реализации `highlight-extension` горазо шире чем то что можно увидеть на текущий момент. Я планрую и дальше развивать эту экосистему добавляя в неё больше приложений, задействуя больше различный технологий, языков программирования и библиотек (по мере изучения). По этому хоть на этом этапе этого не совсем видно, вопрос гибкости для этого монорепо крайне важен.
- Ну и в конце концов это учебный проект цель которого расширение и углубление знаний и навыков. Мне было не так страшно выделить больше времени на настройку проекта, получив в замен больше знаний и навыков.

### Почему e2e тесты вынесены в отдельные npm воркспейсы?

Это стандартный подход при постороении монорепо. К примеру подход где в `/apps` лежит приложение `some-app`, а рядом с ним `some-app-e2e` заложен в Nx CLI.

В монорепо приложения не должны быть зависимы друг от друга. Это может привести к появлению сложно решаемых и отслеживаемых циклических зависимостях. Делает большую систему из многочисленных приложений более хрупкой, так как поломка в одном приложении будет приводить к поломке в другом и так по цепочке.

Именно по этому оставлять e2e тесты внутри одного npm воркспейса с приложением не лучшая идея. Хоть одно приложение не импортирует ничего на прямую из других, оно может посылать запросы на другие и работать на основе результатов полученных из другого приложения. По этому при написании e2e тестов часто приходится импортировать что-то сразу из нескольких приложений.

# Не техническое описание приложений

## highlight-extension

Это браузерное расширения для более удобного и качественного исследования и изучения материалов в интернете.

С помощью этого расширения пользователь может делать заметки (хайлайты) разными цветами внутри любой веб страницы. Снабжать эти заметки (хайлайты) комментариями (ноутсами).

Просматривать краткое содержание страницы благодаря удобной боковой панели в которой отображаются все хайлайты и ноутсы для активной страницы. При клике на хайлайт из боковой панели настроена функция удобной навигации к выбранному хайлайту прямо на веб странице.

Настройки пользователя, возможность добавлять, удалять, менять местами хайлайты и цвета. Быстро включать и отключать расширение.

Просматривать все доступные страницы с хайлайтами и закладками. Возможность мёрджить хайлайты и ноутсы страниц между собой. Менять url страницы в настройках (что бы не терять свои хайлайты и ноутсы при смене адреса веб страницы)

## freq-words

Это переводчик и словарь интегрированный с ИИ. На данный момент реализована только бекенд часть.

Если пользователь вводит слово или устойчивое выражение он получит подробный ответ на подобии context reverso или словаря (синоными, несколько вариантов перевода, примеры) Если пользователь вводит предложение или текст, он получчит простой перевод.

Его ключевая особенность в расширенном функционале закладок. При добавлении слова выраджения в закладки, алгоритм определяет тип слова (лемма[^1] или обычная слово-форма[^2]). Исходя из этого она добавляет заметку не в кучу других, а групирует разные слово-формы вместе с другими слово-формами одного слова[^3]. Так же ведётся подсчёт того, сколько слово-форма была дабавлена в закладки раз.

> [!IMPORTANT]
> Благодаря такому подходу закладки будут выглядить примерно так:
>
> все варианты слова dare - 4\
> dare - 1\
> durst - 2\
> dared - 1
>
> все варианты слова keep - 2\
> keep - 0\
> kept - 2

У такого похода два ключевых преймущества:

1. Таким образом пользователь наглядно будет видеть какие слова ему часто попадаются, но он их каждый раз не запоминает. Это поможет ему определять, то на какие слова нужно уделять особое внимание.

2. Сможет наглядно связывать сразу несколько вариаций одного и того же слова. Это поможет определить что слово встречается часто, просто в разных вариациях, а так же может помочь связать новую вариацию слова с уже знакомой

[^1]: Лемма это ... бла бла.

[^2]: Слово-форма это ... бла бла

[^3]: Слово это ... бла бла

## iam (Identity and Access Management)

Тут находится всё что связанное авторизацией, аутентификацией и управлением пользователем.

Аутентификацие для всех приложений построена именно через это приложение. Именно с помошью него пользователь может войти в аккаунт, зарегистрироватся, поменять пароль, почту, юзернейм и.т.д.
